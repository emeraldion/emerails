Detailed Description

The abstract base class for DB-backed model objects.

Project EmeRails - Codename Ocarina

Copyright (c) 2008, 2017 Claudio Procida http://www.emeraldion.it

Every object of a subclass of ActiveRecord are mapped 1:1 to records of a table in a relational Database. Naming conventions assume that if the ActiveRecord subclass is called MyProduct, database records are stored in a table called my_products. Conversely, if an object of another class is in a relation with your object, it is assumed that a foreign key called my_product_id exists in the other table. Of course this is overridable by setting explicitly $table_name and $foreign_key to a value of your choice.

ActiveRecord::__construct (   $_values = NULL)

Constructs and initializes an ActiveRecord object.

Due to the lack of a static class initialization method, the default constructor is in charge of gathering information about the bound table columns the first time an object is created. Subsequent creations will use the values stored in static class variables. Subclassers don't need to override the constructor. They can in turn override the init method in order to perform custom initialization.

Parameters
valuesColumn values to initialize the object.

Returns the name of the foreign key for this class.

ActiveRecord::save ( )

Requests the receiver to save its data in the bound table.

This method has two distinct effects. If called on an object fetched from the table, it performs an UPDATE SQL statement to update the table data to the new values. If called on an object created programmatically, it performs an INSERT SQL statement, and sets the object's primary key value to the value resulting by the insert.

Returns
This method returns TRUE if the object has been saved successfully.
